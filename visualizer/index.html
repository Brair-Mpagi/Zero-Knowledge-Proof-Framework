Interactive Protocol Walkthrough

Step-by-step mathematical walkthrough of interactive vs. non-interactive (Fiat-Shamir) Schnorr proof protocols.

Phase 1: Setup & Parameters

We work in a prime-order finite field $\mathbb{F}_p$ and an elliptic curve group $\mathbb{G}$ with generator point $g$.

$$\mathbb{F}_p = \{0, 1, 2, ..., p-1\}$$ $$g \in \mathbb{G}$$

The Prover possesses a secret value $x$ (the witness) and publishes a public key $Y$:

$$Y = g^x$$

The goal is to prove knowledge of $x$ without revealing it.

Phase 2: Prover Commitment

The Prover picks a cryptographically secure random value $k \in \mathbb{F}_p$ (called the nonce or blinding factor).

Then, the Prover computes a commitment point $R$ and sends it to the Verifier:

$$k \leftarrow_R \mathbb{F}_p$$ $$R = g^k$$

This hides the secret value but binds the Prover to a random selection for the duration of the proof run.

Phase 3: Challenge Generation

Interactive Protocol

In an interactive setting, the Verifier sends a randomly selected challenge scalar $c \in \mathbb{F}_p$ back to the Prover.

$$c \leftarrow_R \mathbb{F}_p$$

Non-Interactive (Fiat-Shamir)

In a non-interactive setting, the Prover computes the challenge locally by hashing the public parameters and the commitment:

$$c = H(\text{domain} \parallel g \parallel Y \parallel R)$$

This makes the challenge unpredictable but deterministically verifiable.

Phase 4: Prover Response

The Prover computes the response value $s$ using their secret key $x$, the nonce $k$, and the challenge $c$:

$$s = k + c \cdot x \pmod p$$

The Prover sends the response $s$ along with the commitment $R$ as the proof object $\pi = (R, s)$.

Phase 5: Verification Equation

The Verifier recomputes the challenge $c$ (using the hash function or the challenge they sent) and checks if:

$$g^s \stackrel{?}{=} R \cdot Y^c$$

Why does this hold?

By substituting the definition of $s$ and $Y$:

$$g^s = g^{k + c \cdot x} = g^k \cdot (g^x)^c = R \cdot Y^c$$

If the equation holds, the Verifier accepts the proof. Zero knowledge is preserved because $s$ is randomized by $k$, making it mathematically impossible to recover $x$.
